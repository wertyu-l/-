# 用户模块接口文档

## 一、模块基础说明

- 模块基础URL：`/user

- 状态约定：1 代表启用、上线；0 代表禁用、下线

## 二、统一返回结果实体 Result\<T\>

项目所有接口统一使用 Result\<T\> 类封装返回数据，规范接口返回格式，字段说明如下：

|参数名|数据类型|说明|
|---|---|---|
|code|Integer|响应状态码：1=请求成功，0=请求失败|
|msg|String|提示信息，接口请求失败时存放具体错误信息，成功时可返回空或成功提示|
|data|T（泛型）|后端返回的核心业务数据，无返回数据时可为空|

实体类核心代码：

```java
public class Result<T> {
    // 响应码：1成功，0失败
    private Integer code;
    // 响应提示信息
    private String msg;
    // 响应业务数据
    private T data;
}
```

## 三、接口详细说明

### 1\. 新增用户

- 请求地址：`/user`

- 请求方式：POST

#### 请求参数

|参数名|是否必填|参数说明|
|---|---|---|
|id|非必须|用户编号|
|username|必须|用户名|
|password|必须|用户密码|
|role|非必须|用户角色|
|user\_type|非必须|用户类型|
|user\_level|非必须|用户等级|
|department|非必须|所属部门|
|position|非必须|用户职位|
|phone|非必须|联系方式|
|email|非必须|用户邮箱|

#### 返回数据

统一返回 Result 格式，仅返回 code 状态码，无额外业务数据。

### 2\. 删除用户

- 请求地址：`/user`

- 请求方式：DELETE

- 请求参数格式：URL 参数

#### 请求参数

|参数名|是否必填|参数说明|
|---|---|---|
|username|必须|待删除的用户名|

#### 业务规则

处于启用状态（is\_enabled=1）的用户不允许执行删除操作，仅禁用用户可删除。

#### 返回数据

统一返回 Result 格式，仅返回 code 状态码，无额外业务数据。

### 3\. 修改用户

- 请求地址：`/user`

- 请求方式：PUT

#### 请求参数

|参数名|是否必填|参数说明|
|---|---|---|
|id|非必须|用户编号|
|username|非必须|用户名|
|password|非必须|用户密码|
|role|非必须|用户角色|
|user\_type|非必须|用户类型|
|user\_level|非必须|用户等级|
|department|非必须|所属部门|
|position|非必须|用户职位|
|phone|非必须|联系方式|
|email|非必须|用户邮箱|

#### 返回数据

统一返回 Result 格式，包含 code 状态码、data 修改后的用户业务数据。

### 4\. 分页查询用户

- 请求地址：`/user/page`

- 请求方式：GET

#### 请求参数

|参数名|是否必填|参数说明|
|---|---|---|
|username|非必须|用户名，支持模糊查询|
|page|必须|当前查询页码|
|pagesize|必须|每页展示的记录条数|

#### 业务说明

基于 Mybatis 分页插件实现分页查询功能，自动完成数据分页、总条数统计。

#### 返回数据

统一返回 Result 格式，包含 code 状态码、data 分页结果数据（用户列表、总页数、总条数等）。

### 5\. 用户登录

- 请求地址：`/user/login`

- 请求方式：POST

#### 请求参数

|参数名|是否必填|参数说明|
|---|---|---|
|username|必须|登录用户名|
|password|必须|登录密码|

#### 业务逻辑

校验前端传入的用户名和密码，匹配数据库用户数据，校验通过后生成 JWT 令牌并返回。

#### 返回数据

统一返回 Result 格式，包含 code 状态码、data（存储生成的 JWT 登录令牌）。

### 6\. 用户退出登录

- 请求地址：`/user/logout`

- 请求方式：POST

#### 请求参数

无请求参数

#### 返回数据

统一返回 Result 格式，仅返回 code 状态码，无额外业务数据。
